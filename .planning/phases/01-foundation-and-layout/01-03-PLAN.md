<task type="auto">
  <name>Task 1: Deploy to Vercel via CLI</name>
  <files>fintrack-dashboard/vercel.json</files>
  <action>
    1. Check if Vercel CLI is installed: `npx vercel --version`. If not available, install globally: `npm i -g vercel`.

    2. From the `fintrack-dashboard/` directory, run `npx vercel` to link/create the project. Use the fintrack-dashboard directory as the root (not the monorepo root).
       - If prompted for project setup: Framework = Next.js, Root directory = . (already in fintrack-dashboard)
       - This will create a preview deployment

    3. If Vercel CLI requires authentication, create a checkpoint:human-action for the user to run `npx vercel login` first.

    4. After successful link, configure environment variables via CLI:
       ```
       npx vercel env add NEXT_PUBLIC_SUPABASE_URL production
       npx vercel env add NEXT_PUBLIC_SUPABASE_ANON_KEY production
       npx vercel env add SUPABASE_SERVICE_ROLE_KEY production
       npx vercel env add APP_PASSWORD production
       ```
       Note: If CLI env commands require interactive input, instruct user to set env vars in Vercel Dashboard instead (Settings -> Environment Variables).

    5. Deploy to production: `npx vercel --prod`

    6. Record the deployment URL for the verification checkpoint.
  </action>
  <verify>
    <automated>cd fintrack-dashboard && npx vercel ls 2>/dev/null | head -5</automated>
  </verify>
  <done>Vercel project created and deployed. Environment variables configured for production. Production URL accessible.</done>
</task>
