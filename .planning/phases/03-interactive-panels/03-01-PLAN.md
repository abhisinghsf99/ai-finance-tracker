<task type="auto" tdd="true">
  <name>Task 2: Create recurring detection utility, TransactionWithAccount type, and query extension</name>
  <files>
    fintrack-dashboard/src/lib/recurring-detection.ts,

  <behavior>
    - normalizeMerchant: uses merchant_entity_id when available, otherwise lowercases merchant_name, strips .com/.net/Inc/LLC suffixes, trims
    - roundAmount: rounds to nearest cent using Math.round(abs * 100) / 100
    - inferFrequency: calculates median interval between sorted dates; <=9 days = weekly, <=18 = biweekly, <=45 = monthly, else yearly
    - detectRecurring: groups spending transactions (amount > 0) by normalizeMerchant + roundAmount key, filters groups with count >= 3, returns RecurringCharge[] sorted by amount descending
    - RecurringCharge has: merchantName, amount, frequency, lastChargeDate, chargeCount, transactions
    - getTransactionsWithAccounts: fetches transactions with joined account name via Supabase select
  </behavior>
  <action>
    Create `src/lib/recurring-detection.ts` following the algorithm from RESEARCH.md:

    1. Export `RecurringCharge` interface: `{ merchantName: string; amount: number; frequency: 'weekly' | 'biweekly' | 'monthly' | 'yearly'; lastChargeDate: string; chargeCount: number; transactions: Transaction[] }`
    2. Internal `normalizeMerchant(t: Transaction): string` -- prefer merchant_entity_id, fallback to normalized merchant_name (lowercase, strip suffixes, trim)
    3. Internal `roundAmount(amount: number): number`
    4. Internal `inferFrequency(dates: string[]): RecurringCharge['frequency']` -- median interval approach
    5. Export `detectRecurring(transactions: Transaction[]): RecurringCharge[]` -- only process spending (amount > 0), group, filter count >= 3, sort by amount descending
    6. Export `estimateMonthlyTotal(charges: RecurringCharge[]): number` -- weekly * 4.33, biweekly * 2.17, monthly * 1, yearly / 12

    Then extend `src/lib/queries/transactions.ts`:

    7. Add `TransactionWithAccount` type extending Transaction with `account_name: string`
    8. Add `getTransactionsWithAccounts(options?: { limit?: number; offset?: number }): Promise<TransactionWithAccount[]>` -- use Supabase `.select('*, accounts(name)')` to join account name, map result to flatten account_name field

    - Test count threshold: 2 occurrences excluded, 3+ included
    - Test frequency: monthly charges (30-day intervals), weekly charges (7-day intervals)
    - Test sorting: higher amount first
    - Test estimateMonthlyTotal: weekly + monthly mixed correctly
    - Test only spending: negative amounts excluded from detection
