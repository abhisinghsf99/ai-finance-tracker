  <name>Task 1: Implement middleware auth gate and API route</name>
  <files>
    fintrack-dashboard/src/middleware.ts

  </files>
  <behavior>
    - Middleware: unauthenticated GET / returns redirect to /login
    - Middleware: request with valid "fintrack-session" cookie passes through
    - Middleware: GET /login without cookie passes through (no redirect loop)
    - Middleware: GET /api/auth without cookie passes through
    - Middleware: GET /login WITH valid cookie redirects to / (already authenticated)
    - Middleware: GET /_next/static/... is excluded from matcher (no auth check)
    - API: POST /api/auth with correct password returns 200 + sets "fintrack-session" HttpOnly cookie with 30-day maxAge
    - API: POST /api/auth with wrong password returns 401 + { error: "Incorrect password" }
    - API: POST /api/auth with missing body returns 401
  </behavior>
  <action>
    **Step 1 — Implement the middleware (src/middleware.ts):**
    Follow RESEARCH.md Pattern 2 exactly. Key details:
    - Cookie name: `fintrack-session`
    - Matcher: `['/((?!_next/static|_next/image|favicon.ico|fonts/).*)']` — excludes static assets
    - Allow `/login` and `/api/auth` without cookie
    - If authenticated user visits `/login`, redirect to `/`
    - If unauthenticated user visits any other route, redirect to `/login`
    - Do NOT validate the cookie value in middleware (just check existence) — this is a single-user personal app

    **Step 2 — Implement the auth API route (src/app/api/auth/route.ts):**
    Follow RESEARCH.md Pattern 3. Key details:
    - Export an async `POST` function
    - Parse `{ password }` from request body
    - Compare against `process.env.APP_PASSWORD`
    - On match: set `fintrack-session` cookie with value `crypto.randomUUID()`, HttpOnly, secure in production, sameSite: 'lax', maxAge: 30 days, path: '/'
    - On mismatch: return 401 with `{ error: "Incorrect password" }`
    - On missing/malformed body: return 401

    **Step 3 — Implement the test stubs from Wave 0:**
    Replace the `it.todo()` stubs in `middleware.test.ts` and `auth-api.test.ts` with actual tests.

    For middleware tests: Mock `NextRequest` and test the middleware function directly. Verify redirect URLs and NextResponse.next() calls.

    For auth API tests: Mock `process.env.APP_PASSWORD`, call the POST handler with mock NextRequest, verify response status and cookie setting.

    **Anti-patterns to avoid:**
    - Do NOT use JWT, bcrypt, or database sessions — this is a single password comparison
    - Do NOT use NextAuth or any auth library
    - Do NOT hardcode the password — always read from env var
    - Do NOT forget `secure: process.env.NODE_ENV === 'production'` on the cookie (breaks in prod otherwise)
  </action>
  <verify>
    <automated>cd fintrack-dashboard && npx vitest run src/__tests__/middleware.test.ts src/__tests__/auth-api.test.ts --reporter=verbose</automated>
  </verify>
  <done>
    - Middleware redirects unauthenticated requests to /login
    - Middleware passes through authenticated requests
    - Middleware excludes static assets from auth check
    - API returns 200 + cookie on correct password
    - API returns 401 on wrong password
    - All middleware and auth API tests pass
  </done>

    - Full-bleed background with subtle gradient (`bg-gradient-to-br from-background to-muted`)
    - Floating Card centered vertically and horizontally (max-w-sm)
    - App branded as "FinTrack" in the card header (text-2xl font-bold)
    - Single password input (type="password", autoFocus, placeholder="Enter password")
    - "Sign In" button using shadcn Button component (full width)
    - On submit: POST to `/api/auth` with `{ password }`
    - On success (200): `router.push('/')` then `router.refresh()`
    - On error (401): set shake state true, show "Incorrect password" in destructive color, clear shake after 500ms

    **Step 2 — Add shake animation to globals.css:**
    Add a CSS keyframe animation for the shake effect:
    ```css
    @keyframes shake {
      0%, 100% { transform: translateX(0); }
      25% { transform: translateX(-8px); }
      50% { transform: translateX(8px); }
      75% { transform: translateX(-4px); }
    }
    ```
    And a utility class `.animate-shake { animation: shake 0.5s ease-in-out; }` — or add it via the `@theme inline` block if Tailwind 4 supports custom animations there.

    **Design details (per Copilot aesthetic from CONTEXT.md):**
    - Card should have soft shadow, rounded corners (uses --radius from theme)
    - Login page should feel warm and approachable, not corporate

    - Reference design-system/MASTER.md for color/spacing guidance

    - Do NOT add lockout logic (user decision: no lockout)
    - Do NOT add "forgot password" or any recovery flow (single password, no recovery needed)
