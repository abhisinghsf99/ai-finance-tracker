  <action>
    **Step 1 — Generate design system FIRST (per user decision):**
    Run the UI UX Pro Max skill to generate the design system:
    ```bash
    python3 skills/ui-ux-pro-max/scripts/search.py "fintech dashboard copilot warm approachable teal" --design-system --persist -p "FinTrack"
    ```
    Note: The `skills/` directory is at `/Users/abhisingh/my-os/skills/ui-ux-pro-max/scripts/search.py`. Move the generated `design-system/` folder into `fintrack-dashboard/` after scaffold.

    **Step 2 — Scaffold the project:**
    ```bash
    npx create-next-app@15 fintrack-dashboard --typescript --tailwind --eslint --app --src-dir --use-npm
    ```
    After scaffold, verify Next.js version >= 15.2.3 (CVE-2025-29927 fix):
    ```bash
    cd fintrack-dashboard && npx next --version
    ```

    **Step 3 — Initialize shadcn/ui:**
    ```bash
    npx shadcn@latest init
    ```
    Accept defaults (New York style, zinc base color — we override colors anyway). Then add Phase 1 components:
    ```bash
    npx shadcn@latest add button input card separator skeleton sheet
    ```

    **Step 4 — Install Satoshi font:**
    Download Satoshi font files (woff2) from Fontshare and place in `public/fonts/`:
    - Satoshi-Regular.woff2
    - Satoshi-Medium.woff2
    - Satoshi-Bold.woff2

    If Satoshi files are not downloadable via CLI, use General Sans as fallback (same neo-grotesque family, same source). Configure in layout.tsx using `next/font/local`:

    ```typescript
    const satoshi = localFont({
      src: [
        { path: '../../public/fonts/Satoshi-Regular.woff2', weight: '400', style: 'normal' },
        { path: '../../public/fonts/Satoshi-Medium.woff2', weight: '500', style: 'normal' },
        { path: '../../public/fonts/Satoshi-Bold.woff2', weight: '700', style: 'normal' },
      ],
      variable: '--font-satoshi',
      display: 'swap',
    })
    ```

    **Step 5 — Configure theme in globals.css:**
    Replace the default globals.css with the teal/cyan color system from RESEARCH.md Pattern 4. Key values:
    - Primary: `hsl(175 70% 41%)` light / `hsl(175 70% 50%)` dark
    - Background: `hsl(0 0% 100%)` light / `hsl(222 47% 5%)` dark
    - Radius: `0.75rem` (rounded cards per Copilot aesthetic)
    - Font: `--font-sans: var(--font-satoshi), system-ui, sans-serif` inside `@theme inline`

    Reference the design system MASTER.md output for any additional color/typography decisions.

    **Step 6 — Create ThemeProvider component:**
    Create `src/components/theme-provider.tsx` as a "use client" wrapper around NextThemesProvider (see RESEARCH.md Pattern 6).

    **Step 7 — Configure root layout.tsx:**
    - Import Satoshi font via `next/font/local`
    - Wrap children in ThemeProvider with `defaultTheme="dark"`, `enableSystem`, `disableTransitionOnChange`
    - Add `suppressHydrationWarning` to `<html>` tag
    - Set `className={satoshi.variable}` on `<html>`
    - Body: `className="font-sans antialiased"`

    **Step 8 — Create temporary root page:**
    Replace the default page.tsx with a simple Server Component that shows "FinTrack" centered, confirming the app renders with the correct font and theme. Keep it as a Server Component (no "use client").

    **Step 9 — Create .env.local:**
    ```
    APP_PASSWORD=changeme
    NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
    SUPABASE_SERVICE_ROLE_KEY=your-service-role-key
    ```
    Note: SUPABASE_SERVICE_ROLE_KEY is intentionally NOT prefixed with NEXT_PUBLIC_ — this keeps it server-only.

    **Anti-patterns to avoid:**
    - Do NOT create tailwind.config.js — Tailwind 4 uses @theme inline in CSS
    - Do NOT use "use client" on page.tsx files
    - Do NOT use Inter, Roboto, or Arial fonts (user decision: distinctive typography)
    - Do NOT install next-themes separately (shadcn/ui init installs it)

  <action>
    **Step 1 — Install Supabase client:**
    ```bash
    cd fintrack-dashboard && npm install @supabase/supabase-js
    ```

    **Step 2 — Create server Supabase client:**
    Create `src/lib/supabase/server.ts` per RESEARCH.md Pattern (Supabase Server Client):
    - Export `createServerSupabase()` function
    - Read `NEXT_PUBLIC_SUPABASE_URL` and `SUPABASE_SERVICE_ROLE_KEY` from `process.env`
    - Throw with prefixed error `[supabase] Missing ...` if either is missing (per project convention of prefixed console logging)
    - Pass `{ auth: { persistSession: false } }` to createClient (server-side, no session persistence needed)

    **Step 3 — Install test dependencies:**
    ```bash
    npm install -D vitest @testing-library/react @testing-library/jest-dom @vitejs/plugin-react jsdom
    ```

    **Step 4 — Create vitest.config.ts:**
    ```typescript
    import { defineConfig } from 'vitest/config'
    import react from '@vitejs/plugin-react'
    import path from 'path'

    export default defineConfig({
      plugins: [react()],
      test: {
        environment: 'jsdom',
        setupFiles: [],
        globals: true,
      },
      resolve: {
        alias: {
          '@': path.resolve(__dirname, './src'),
        },
      },
    })
    ```

    **Step 5 — Create test stub files (Wave 0 from VALIDATION.md):**
    Create these 4 test files with `describe` blocks and `it.todo()` placeholders:

    - `src/__tests__/middleware.test.ts`: Stubs for "redirects unauthenticated requests to /login", "allows requests with valid session cookie", "allows /login and /api/auth without cookie"
    - `src/__tests__/auth-api.test.ts`: Stubs for "returns 401 for wrong password", "returns 200 and sets cookie for correct password", "returns 401 for missing password"
    - `src/__tests__/sidebar.test.ts`: Stubs for "renders 4 navigation items", "highlights active page", "renders FinTrack branding"
    - `src/__tests__/theme-toggle.test.ts`: Stubs for "renders sun icon in dark mode", "renders moon icon in light mode", "toggles theme on click"

    Each stub file should import `describe` and `it` from `vitest`. Use `it.todo('description')` for each test case. These will be implemented by the plans that create the actual components.

    **Step 6 — Add test script to package.json:**
    Add `"test": "vitest run --reporter=verbose"` to the scripts section if not already present.
