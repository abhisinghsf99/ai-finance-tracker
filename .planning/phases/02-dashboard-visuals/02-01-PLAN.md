  </files>
  <action>
    1. Install shadcn chart component:
       ```bash
       cd fintrack-dashboard && pnpm dlx shadcn@latest add chart
       ```
       This installs recharts and creates components/ui/chart.tsx. If react-is version conflict occurs, add to package.json:
       ```json
       "pnpm": { "overrides": { "react-is": "^19.1.0" } }
       ```
       Then re-run pnpm install.

    2. Add three new query functions to `src/lib/queries/transactions.ts` (append, do not modify existing functions):

       - `getTrailingMonthlySpending(months = 6)`: Loops months-1 down to 0, calls existing `getMonthlySpending()` for each month. Returns `{ month: string (short name like "Jan"), total: number, count: number }[]` sorted oldest-first. Use `Promise.all()` for parallel fetching.

       - `getCategorySpending(yearMonth: string)`: Queries transactions for given month, filters to spending only (amount > 0), groups by `category_primary`, returns `{ category: string, total: number, count: number }[]` sorted by total descending. Null categories map to "OTHER".

       - `getTransactionsByCategory(yearMonth: string, category: string)`: Queries transactions for month + category_primary match, spending only (amount > 0), ordered by date descending. Returns `Transaction[]`.

    3. Create `src/lib/chart-colors.ts` with a `CATEGORY_COLORS` record mapping Plaid category_primary values to HSL color strings. Include at least 13 entries covering: FOOD_AND_DRINK, TRANSPORTATION, ENTERTAINMENT, SHOPPING, GENERAL_MERCHANDISE, RENT_AND_UTILITIES, PERSONAL_CARE, TRAVEL, GENERAL_SERVICES, LOAN_PAYMENTS, TRANSFER_OUT, MEDICAL, OTHER. Use muted harmonious HSL values that work well on dark backgrounds. Export a `getCategoryColor(category: string)` function that falls back to OTHER color.
  </action>
  <verify>
    <automated>cd fintrack-dashboard && pnpm build 2>&1 | tail -5</automated>
  </verify>
  <done>Recharts installed via shadcn chart, 3 new query functions exported from transactions.ts, chart-colors.ts has 13+ color entries with getCategoryColor fallback. Build passes.</done>
