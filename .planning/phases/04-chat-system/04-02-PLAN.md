  <action>
Create all 5 chat UI components. All are "use client" components.

1. **typing-indicator.tsx** -- Export `TypingIndicator` component:
   - Three animated dots in a muted bubble (same style as assistant bubble)
   - Use CSS animation: each dot has a bounce/pulse with staggered delay (0ms, 150ms, 300ms)
   - Dots: `h-2 w-2 rounded-full bg-muted-foreground/50`
   - Container matches assistant bubble style: `bg-muted rounded-2xl px-4 py-3`

2. **suggestion-chips.tsx** -- Export `SuggestionChips` component:
   - Props: `{ onSelect: (text: string) => void }`
   - Import `SUGGESTION_CHIPS` from `@/lib/chat-config`
   - Render centered greeting: "Ask me anything about your finances" in `text-muted-foreground`
   - Below greeting: flex-wrap container with 4 chip buttons
   - Each chip: `border border-border rounded-full px-4 py-2 text-sm hover:bg-muted transition-colors cursor-pointer`
   - On click: call `onSelect` with the chip text

3. **chat-input.tsx** -- Export `ChatInput` component:
   - Props: `{ onSend: (text: string) => void, disabled: boolean }`
   - Local state for input text
   - Form with flex layout: text input + send button
   - Input: `flex-1 bg-transparent border-none outline-none text-sm placeholder:text-muted-foreground` inside a `border border-border rounded-full px-4 py-2 bg-muted/50` container
   - Placeholder: "Ask about your finances..."
   - Send button: `Send` icon from lucide-react (or `ArrowUp`), hidden when input is empty, `bg-primary rounded-full h-8 w-8 flex items-center justify-center`
   - Submit on Enter key (not Shift+Enter) and on button click
   - Clear input after send
   - Disable input and button when `disabled` is true
   - Minimum touch target 44x44px for send button (per skill accessibility rules)

4. **chat-view.tsx** -- Export `ChatView` component:
   - Props: `{ onClose: () => void }`
   - Use `useChat` from `@ai-sdk/react` -- this auto-posts to `/api/chat`
   - Destructure: `{ messages, sendMessage, status, setMessages, error }`
   - Derive: `isLoading = status === 'streaming' || status === 'submitted'`
   - Derive: `isEmpty = messages.length === 0`
   - Full-screen fixed overlay: `fixed inset-0 z-50 bg-background flex flex-col`
   - **Header**: flex row with:
     - Back button (ArrowLeft icon from lucide) that calls `onClose` -- accessible with aria-label "Close chat"
     - Title: "FinTrack Chat" centered
     - New Chat button (RotateCcw or RefreshCw icon from lucide) that calls `setMessages([])` -- accessible with aria-label "New chat"
     - Header styling: `border-b border-border px-4 py-3`
   - **Messages area**: `flex-1 overflow-y-auto px-4 py-4 space-y-3`
     - If `isEmpty`: render `SuggestionChips` centered vertically with `flex-1 flex items-center justify-center`
     - Otherwise: map `messages` through `ChatMessage` component (skip tool-invocation role messages in render)
     - If `isLoading` and last message is not from assistant (submitted state): show `TypingIndicator`
     - Auto-scroll to bottom on new messages using a ref + useEffect
   - **Error display**: if `error`, show a subtle red banner above input: "Something went wrong. Try again."
   - **Input area**: `border-t border-border px-4 py-3 pb-[env(safe-area-inset-bottom,12px)]`
     - Render `ChatInput` with `disabled={isLoading}`
     - `onSend` calls `sendMessage({ parts: [{ type: 'text', text }] })`
   - When `onClose` is called, conversation is automatically cleared (component unmounts, useChat state is destroyed -- fresh each time per CONTEXT.md)

5. **chat-fab.tsx** -- Export `ChatFAB` component:
   - Local state: `isOpen` boolean (default false)
   - Also accept an optional `externalOpen` prop and `onOpenChange` callback for mobile nav integration
   - Use `useEffect` to sync `externalOpen` into `isOpen` when it changes
   - When `isOpen` is true: render `ChatView` with `onClose` that sets `isOpen = false` and calls `onOpenChange?.(false)`
   - When `isOpen` is false: render floating button:
     - `fixed bottom-20 right-4 md:bottom-6 md:right-6 z-40` (bottom-20 on mobile to sit above bottom nav, bottom-6 on desktop)
     - `h-14 w-14 rounded-full shadow-lg shadow-primary/25`
     - `bg-primary hover:bg-primary/90 text-primary-foreground`
     - `MessageSquare` icon from lucide-react, `h-6 w-6`
     - `aria-label="Open chat"`
     - Use Button component from `@/components/ui/button` with `size="icon"`
   - Export a `useChatOpen` pattern or expose `openChat` function for mobile nav to call
